# Document Roadmap

> **Overview of the Engineering Playbook structure**

This roadmap explains how the Playbook is organized and how the documents relate to each other.

---

# Engineering Lifecycle

The Playbook follows the complete engineering lifecycle.

```text
Learn
   ↓
Build
   ↓
Validate
   ↓
Deploy
   ↓
Operate
   ↓
Recover
```

---

# Foundation

The Foundation documents introduce the platform architecture and engineering concepts.

| Documents | Description |
|-----------|-------------|
| 00–06 | Platform overview, architecture, glossary and engineering decisions |

---

# Build & Rebuild

These documents describe how to reproduce the platform from an empty workstation.

| Documents | Description |
|-----------|-------------|
| 10–20 | Workstation, Docker, CI/CD, GitOps, Kubernetes and complete rebuild |

---

# Operations

Operational documentation for the running platform.

| Documents | Description |
|-----------|-------------|
| 30–35 | Daily operations, validation, incident response and recovery |

---

# Troubleshooting

Structured troubleshooting documentation.

| Documents | Description |
|-----------|-------------|
| 40–45 | Git, CI/CD, ArgoCD, Kubernetes and Lessons Learned |

---

# Recommended Reading Order

New engineers should follow this sequence.

```text
Foundation
      ↓
Build
      ↓
Operations
      ↓
Troubleshooting
```
