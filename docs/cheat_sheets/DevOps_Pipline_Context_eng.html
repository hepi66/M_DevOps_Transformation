DevOps Pipeline Context Cheat-Sheet

Internal Quick Overview: From Source Code to Running Image

1. Local Dev (Git)

Serves as the foundation for decentralized version control directly on the developer's workstation. It captures every code change incrementally and allows working in isolated feature branches. Local commits form the initial starting point for the entire deployment cycle.

2. GitHub Remote Repository

Acts as the central cloud platform and Single Source of Truth for the entire project. In addition to hosting the raw code, it manages the bridge to automation via webhooks and event triggers and safeguards code quality through Pull Request workflows.

3. GitHub Projects (Scrum)

Handles the structuring and tracking of the agile product backlog within sprints. It translates high-level technical milestones into actionable epics and user stories, visually capturing dependencies and managing sprint planning natively alongside the code.

4. GitHub Actions (CI Pipeline)

Responsible for Continuous Integration (CI). Automatically triggered upon any Git push event, this engine compiles code, runs automated test suites, validates changes, and upon successful execution, builds the immutable Docker container image.

5. Docker Packaging & GHCR (Registry)

Docker encapsulates the application and all its dependencies into a lightweight, portable container image. This final artifact is securely pushed to and versioned within the GitHub Container Registry (GHCR), ready to be pulled by the cluster environment.

6. Argo CD (GitOps Reconciliation)

The core engine of Continuous Delivery (CD) using GitOps. It continuously reconciles the declarative desired state stored in the Git repository against the live state of the Kubernetes cluster, instantly auto-correcting or flagging configuration drifts.

7. K3s Kubernetes (Target Cluster)

A highly lightweight and resource-efficient container orchestration runtime environment. It runs the application containers fetched from the registry inside pods, handles routing and scaling, and ensures the resilient operation of the MVP platform.
