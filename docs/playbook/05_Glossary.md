# Glossary

## Purpose

This glossary defines the most important technical terms used throughout the M-DevOps Playbook.

The objective is to provide a common vocabulary for engineers working with the platform.

This document should be used whenever an unfamiliar term appears in the Playbook.

---

# A

## Application

The software being delivered and operated by the platform.

In this project:

```text id="n7v3pk"
Streamlit Application

app.py
```

The application is packaged into a container and deployed to Kubernetes.

---

## ApplicationSet

A Kubernetes Custom Resource provided by ArgoCD.

Purpose:

```text id="v2m8qx"
Automated Application Management

GitOps Scalability

Multi-Application Deployment
```

Validation Example:

```text id="j4q7wd"
ApplicationSet CRD Installed
```

Related Documents:

```text id="k9v2pr"
15_GitOps_and_ArgoCD_Guide.md

19_ArgoCD_Bootstrap_Guide.md

43_ArgoCD_Troubleshooting.md
```

---

## ArgoCD

A GitOps platform used to synchronize Kubernetes resources from Git repositories.

Role:

```text id="a8m4pk"
Git
        ↓
ArgoCD
        ↓
Kubernetes
```

ArgoCD continuously reconciles desired and actual state.

Related Documents:

```text id="s5q9mx"
15_GitOps_and_ArgoCD_Guide.md

19_ArgoCD_Bootstrap_Guide.md

43_ArgoCD_Troubleshooting.md
```

---

# B

## Bandit

A Python security analysis tool.

Purpose:

```text id="d3v8tr"
Static Security Scanning
```

Used during CI/CD.

Bandit helps identify potential security issues before deployment.

---

## Bootstrap

The process of creating an operational platform from an empty environment.

Examples:

```text id="p6m2wd"
Kubernetes Bootstrap

ArgoCD Bootstrap
```

Related Documents:

```text id="u4q7vn"
18_Kubernetes_Bootstrap_Guide.md

19_ArgoCD_Bootstrap_Guide.md
```

---

# C

## CI/CD

Continuous Integration and Continuous Delivery.

Purpose:

```text id="x8m3pk"
Automated Validation

Automated Build

Automated Delivery
```

Implemented using GitHub Actions.

---

## Commit

A recorded change in Git.

Example:

```powershell id="z2q9mx"
git commit -m "description"
```

Commits create the project history.

---

## Container

A portable runtime package containing:

```text id="r5v7pk"
Application

Dependencies

Runtime Configuration
```

Containers provide consistent execution environments.

---

## Container Image

The immutable artifact created during Docker build.

Example:

```text id="w7m2qx"
m-devops-transformation
```

Container images are stored in GHCR.

---

## CRD (Custom Resource Definition)

A Kubernetes mechanism that extends the Kubernetes API.

Example:

```text id="m4q8wd"
ApplicationSet CRD
```

CRDs allow platforms such as ArgoCD to introduce new resource types.

---

# D

## Deployment

A Kubernetes resource responsible for managing Pods.

Purpose:

```text id="y2v6pk"
Desired Replica Count

Updates

Rollouts

Recovery
```

Deployments are the primary workload resource.

---

## Docker

A containerization platform used to package applications.

Key File:

```text id="c9m4tr"
Dockerfile
```

Docker produces container images used for deployment.

---

## Dockerfile

The build specification for a container image.

Defines:

```text id="u8q2vx"
Base Image

Dependencies

Startup Commands
```

The Dockerfile is one of the most important platform artifacts.

---

# G

## GHCR

GitHub Container Registry.

Purpose:

```text id="s4m9pk"
Container Image Storage

Artifact Distribution
```

Example:

```text id="n5q3wd"
ghcr.io/<owner>/m-devops-transformation
```

GHCR stores deployment artifacts generated by CI/CD.

---

## Git

A distributed version control system.

Used for:

```text id="r7v2mx"
Source Code

Documentation

Infrastructure Definitions
```

Git is the foundation of the platform.

---

## GitHub Actions

GitHub's automation platform.

Used for:

```text id="f2m8pk"
CI/CD

Testing

Container Build

Publication
```

Pipeline Definition:

```text id="g9q4wd"
.github/workflows/ci.yml
```

---

## GitOps

An operational model where Git becomes the source of truth.

Flow:

```text id="v6m3tr"
Git
        ↓
ArgoCD
        ↓
Kubernetes
```

Infrastructure changes are managed through Git rather than manual cluster modifications.

---

## Golden Path

The validated implementation route documented by the Playbook.

Purpose:

```text id="u3v8pk"
Repeatability

Reliability

Consistency
```

The Golden Path represents the recommended engineering workflow.

---

# I

## ImagePullBackOff

A Kubernetes error indicating that a container image cannot be retrieved.

Common Causes:

```text id="q8m2vx"
Missing Image

Invalid Tag

Registry Access Issue
```

Related Document:

```text id="x4q7pk"
44_Kubernetes_Troubleshooting.md
```

---

# K

## Kubernetes

A container orchestration platform.

Responsible for:

```text id="n2m9wd"
Scheduling

Deployment

Scaling

Runtime Operations
```

Kubernetes runs the platform workloads.

---

## Kustomization

A configuration file used by Kustomize.

Purpose:

```text id="r5q3vn"
Resource Aggregation

Environment Customization
```

Typically named:

```text id="t7m8pk"
kustomization.yml
```

---

## Kustomize

A Kubernetes configuration management tool.

Used to organize and compose Kubernetes manifests.

---

# M

## Manifest

A YAML definition describing a Kubernetes resource.

Examples:

```text id="p3v7mx"
Deployment

Service

Namespace

Application
```

Manifests define desired platform state.

---

# N

## Namespace

A logical isolation boundary within Kubernetes.

Example:

```text id="v8m4pk"
argocd
```

Namespaces organize workloads and resources.

---

# P

## Pipeline

The automated sequence of validation and delivery activities.

Platform Pipeline:

```text id="w2q9vd"
Ruff
↓
Bandit
↓
pytest
↓
Docker Build
↓
GHCR Publish
```

Implemented through GitHub Actions.

---

## Pod

The smallest deployable unit in Kubernetes.

A Pod contains one or more containers.

Pods execute application workloads.

---

# R

## ReplicaSet

A Kubernetes resource responsible for maintaining the desired number of Pods.

Managed automatically by Deployments.

---

## Root Application

The primary GitOps entry point.

Repository Location:

```text id="q7m4pk"
k8s/apps/root-app.yml
```

Purpose:

```text id="v4q8mx"
Bootstrap GitOps

Register Applications

Control Deployment Hierarchy
```

---

## Ruff

A Python linting and code quality tool.

Purpose:

```text id="g6m3tr"
Static Code Analysis
```

Executed during CI/CD validation.

---

# S

## Service

A Kubernetes networking resource.

Purpose:

```text id="n9q2wd"
Expose Applications

Provide Stable Access
```

Services route traffic to Pods.

---

## Source Of Truth

The authoritative definition of system state.

Within GitOps:

```text id="w8m4pk"
Git Repository
```

is the source of truth.

---

# T

## Troubleshooting

The structured process of diagnosing and resolving issues.

Preferred Workflow:

```text id="y3q7vx"
Observe
↓
Collect Evidence
↓
Identify Cause
↓
Apply Fix
↓
Validate
```

---

# V

## Validation

The process of verifying platform health and correctness.

Platform Scripts:

```text id="p5m8tr"
verify_cluster.ps1

verify_gitops.ps1

verify_pods.ps1

verify_all.ps1
```

Validation is a foundational platform practice.

---

## Virtual Environment (venv)

An isolated Python environment.

Purpose:

```text id="k2q9wd"
Dependency Isolation

Reproducible Development
```

Created locally for development work.

---

# Y

## YAML

A human-readable configuration format.

Widely used for:

```text id="r4m7pk"
Kubernetes Manifests

GitOps Definitions

Configuration Files
```

Most infrastructure definitions in the platform are YAML-based.
